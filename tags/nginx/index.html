Nginx auth_basic

Доброго времени суток! Намедни столкнулся с не очень очевидным поведеним Nginx при использовании auth_basic. Исходные данные - есть один сервер с Nginx который работает как прокси для другого сервера а apache.

server {
  listen       80;
  server_name  host1;

  location / {
      proxy_pass        http://host2;
  }

Наша задача закрыть определенный URL логином:паролем через auth_basic. Мы делаем это так:

server {
  listen       80;
  server_name  host1;

  location / {
      proxy_pass        http://host2;
  }

  location /admin {
      auth_basic           "admin zone";
      auth_basic_user_file /path/to/passwd;
  }

И обнаруживаем что после успешной авторизации мы получаем 404 ошибку. А все дело в том что Nginx пытается после авторизации получить /admin с того же хоста где он сам находится, а не с того куда мы проксируем запрос. Поэтому необходимо сделать следующее:

server {
  listen       80;
  server_name  host1;

  location / {
      proxy_pass        http://host2;
  }

  location /admin {
      auth_basic           "admin zone";
      auth_basic_user_file /path/to/passwd;
      location /admin {
            proxy_pass        http://host2;
    }

  }

Вот такая вот неожиданная проблема и примерно такое же неожиданное решение. =)
